
### 阶段一：破冰与奠基 —— 纯色体积雾块 (基础框架)
**目标：** 在 URP 中跑通光线步进（Raymarching），能在场景里放一个有体积感的“纯色雾块”。
**可用场景：** 毒气陷阱、魔法阵的光晕、简单的局部雾气。

*   **步骤 1：URP 自定义渲染管线接入**
    *   学习并编写一个 `ScriptableRendererFeature`（URP后处理的核心）。
    *   实现一个全屏的 Blit Shader，确保你能用代码改变整个屏幕的颜色。
*   **步骤 2：基础光线步进 (Raymarching)**
    *   在 Shader 中编写 Raymarching 循环。
    *   实现**射线与包围盒（AABB）的相交检测**。让渲染只发生在你指定的 Box 范围内（这是实现“地上云块”的基础）。
*   **步骤 3：深度剔除与软交界 (Soft Intersection)**
    *   读取 URP 的深度图（Depth Texture）。
    *   当雾块与地形或建筑穿插时，利用深度差做 Alpha 渐变，消除生硬的“切边”。
*   **步骤 4：基础参数暴露**
    *   在 C# 脚本中暴露颜色（Color）和基础密度（Density）参数，传递给 Shader。

### 阶段二：雕刻与塑形 —— 静态体积云 (形态学)
**目标：** 引入 3D 噪声，把纯色的雾块“雕刻”成云朵的形状。
**可用场景：** 静态的天空云层、漂浮的云朵岛屿。

*   **步骤 1：生成 3D 噪声 (Compute Shader)**
    *   学习 Compute Shader。编写代码生成 3D Perlin 噪声和 Worley 噪声。
    *   将生成的噪声保存为 `Texture3D` 资产。
*   **步骤 2：噪声采样与密度映射**
    *   在 Raymarching 循环中，根据当前空间坐标采样 3D 噪声图。
    *   用噪声值减去/乘以基础密度，云的边缘就会变得蓬松、不规则。
*   **步骤 3：高度渐变 (Altitude Gradient)**
    *   引入高度控制：云的底部通常是平坦的，顶部是蓬松的。
    *   通过 Y 轴坐标映射一个渐变曲线，控制不同高度的云密度。
*   **步骤 4：基础天气图 (Weather Map)**
    *   引入一张 2D 贴图（从上往下投射）。
    *   黑色代表无云，白色代表有云，实现宏观上的云层分布控制。

### 阶段三：追光者 —— 真实光照与阴影 (光学模型)
**目标：** 赋予云层真实的体积光感、透射感和阴影。
**可用场景：** 影视级截图、高画质慢节奏游戏中的逼真天空。

*   **步骤 1：光线衰减 (Beer-Lambert Law)**
    *   在主 Raymarching 循环内部，向着太阳方向再发射一条短射线（Light Marching）。
    *   计算光线穿透云层时的衰减，实现云层内部的“体积阴影”（暗部）。
*   **步骤 2：相位函数 (Henyey-Greenstein)**
    *   实现 HG 相位函数。这能让云在背对太阳时产生漂亮的**银边（Silver Lining）**，而在正对太阳时产生高光透射。
*   **步骤 3：多重散射近似 (Multiple Scattering)**
    *   叠加几个不同衰减系数的光照计算，模拟光在云内部的多次弹射。解决云层暗部“死黑”的问题，让云变得通透。
*   **步骤 4：云投射阴影 (Cloud Shadows)**
    *   将云的密度图投影到地面的 Shader 中，或者通过修改 URP 的 Main Light Shadow 逻辑，让云在地面上投射出斑驳的阴影。

### 阶段四：性能救星 —— 商业级优化 (工程学)
**目标：** 把帧率从 10 帧提升到 60 帧，达到可以实际发布游戏的标准。
**可用场景：** 任何需要兼顾画质与性能的 3D 游戏。

*   **步骤 1：蓝噪声抖动 (Blue Noise Dithering)**
    *   引入 2D 蓝噪声贴图，偏移每根射线的起始步进位置。
    *   这可以让你把步进次数（Steps）从 128 次降到 32 次，虽然画面会变噪，但性能大幅提升。
*   **步骤 2：降采样渲染 (Half-Resolution)**
    *   修改 Render Feature，将云渲染到一张只有屏幕长宽一半（甚至四分之一）的 RenderTexture 上。
    *   然后再将其放大（Upscale）并叠加到主屏幕上。
*   **步骤 3：时序抗锯齿/重投影 (TAA / Temporal Reprojection)**
    *   这是**最难也是最核心**的一步。
    *   利用相机的运动矢量（Motion Vectors）和上一帧的画面，混合当前帧的噪点画面。把蓝噪声带来的噪点抹平，实现低性能消耗下的丝滑画质。

### 阶段五：随心所欲 —— 极致自定义与 UI (工具化)
**目标：** 实现你的核心诉求：想捏成什么样就捏成什么样，并且有一个好用的面板。
**可用场景：** 作为通用插件上架 Asset Store，供各种美术人员使用。

*   **步骤 1：SDF（有向距离场）接入**
    *   编写工具，将任意 3D Mesh（比如兔子模型）体素化，烘焙成 3D SDF 纹理。
    *   在 Shader 中读取 SDF，让云完全按照模型的形状生成（取代或叠加原来的 Box 限制）。
*   **步骤 2：动态滚动与流向图 (Flow Map)**
    *   给噪声坐标加上时间变量（Time），让云飘动起来。
    *   引入 2D Flow Map，让云可以像漩涡一样旋转，或者沿着特定轨迹流动。
*   **步骤 3：Inspector 面板与 2D 预览**
    *   编写自定义的 Editor 脚本（CustomEditor）。
    *   利用 Compute Shader 实时生成当前参数下的 2D 切片预览图，显示在 Inspector 面板上，做到“所见即所得”。

### 阶段六：画龙点睛 —— 动态交互与特效 (次世代特性)
**目标：** 增加流体扰动、局部光源等高级特性。
**可用场景：** 魔法战斗、飞机穿云等强互动场景。

*   **步骤 1：相机穿云处理 (Camera Penetration)**
    *   处理相机进入云层内部时的逻辑，避免近裁剪面导致的“切面穿帮”。
    *   添加全屏的雾气浓度增加效果。
*   **步骤 2：局部光源支持 (Local Lights)**
    *   将场景中的 Point Light（点光源）数据传入 Shader。
    *   在光照计算中加入点光源的影响，实现云层内部的闪电、爆炸照亮云层的效果。
*   **步骤 3：伪流体交互 (Pseudo-Fluid Interaction)**
    *   真正的 3D 流体太卡。我们可以采用**位置球体排斥（Spherical Repulsion）**。
    *   当角色或飞机（带有特定脚本）穿过云时，将其坐标传给 Shader，在采样噪声时将该坐标周围的密度强制降低（推开云雾），模拟气流扰动。
